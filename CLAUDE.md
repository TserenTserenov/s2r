# Инструкции для Claude Code

## Принципы работы

### First Principles Framework (FPF)

Используй подход **FPF** — мышление от первых принципов для архитектурных решений:

1. Разложи проблему до базовых элементов
2. Определи фундаментальные ограничения и возможности
3. Собери решение с нуля, исходя из первых принципов

### ADI Цикл

Применяй цикл **ADI** (Analyze → Design → Implement):

1. **Analyze** — пойми контекст и требования
2. **Design** — спроектируй решение от первых принципов
3. **Implement** — реализуй минимально достаточное решение

## Перед архитектурной работой

**Всегда загружай** `.fpf/FPF-Spec.md` перед работой над архитектурой:

```
Загрузи и примени спецификацию .fpf/FPF-Spec.md для архитектурных решений
```

Это обеспечит согласованность архитектурных решений с принятыми стандартами проекта.

## Работа с SRD

При работе с этим хранилищем:

1. Размещай документы в соответствующие папки матрицы SRD (системы × роли)
2. Используй YAML frontmatter в документах
3. Следуй минимализму — достаточно, но не избыточно
4. Документируй решения от первых принципов

## Формат документа

Используй шаблон:

```yaml
---
type: [concept|spec|guide|decision]
status: [draft|review|active|archived]
family: [system-role, например: "2.2"]
created: YYYY-MM-DD
---

# Название документа

Содержание...
```
