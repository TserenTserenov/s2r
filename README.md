# SRD Template

Шаблон хранилища знаний по методу **SRD (Systems-Roles-Dimensions)** — структурированная организация проектных знаний через матрицу систем, ролей и измерений.

## Суть метода SRD

SRD — это метод организации знаний о проекте через трёхмерную структуру:

**3 Системы × 3 Роли = 9 ячеек матрицы**

### Три системы

1. **Management (Управление)** — система управления проектом
2. **Supersystem (Надсистема)** — внешняя среда и контекст
3. **Target System (Целевая система)** — основной продукт
4. **Creation System (Система создания)** — процессы разработки

### Три роли (измерения)

1. **Meaning (Смысл)** — ценности, цели, требования
2. **Architecture (Архитектура)** — структура, компоненты, связи
3. **Operations (Операции)** — процессы, действия, практики

### Матрица 3×3

```
                 Meaning          Architecture      Operations
Management       Цели проекта     Структура управл. Процессы управл.
Supersystem      Контекст рынка   Окружение         Взаимодействие
TargetSystem     Требования       Архитектура       Реализация
CreationSystem   Принципы разр.   Инструменты       Методология
```

## Структура проекта

```
srd-template/
├── README.md                    # Описание метода
├── CLAUDE.md                    # Инструкции для Claude Code + FPF
├── CONTRIBUTING.md              # Правила участия
├── .fpf/                        # First Principles Framework
│   └── INDEX.md                 # Индекс фундаментальных принципов
└── content/                     # Контент по методу SRD
    ├── 0.Management/            # Система управления
    │   ├── 0.1.Meaning/         # Смысл: цели, миссия
    │   ├── 0.2.Architecture/    # Архитектура: оргструктура
    │   └── 0.3.Operations/      # Операции: процессы управления
    ├── 1.Supersystem/           # Надсистема
    │   ├── 1.1.Meaning/         # Смысл: контекст, рынок
    │   ├── 1.2.Architecture/    # Архитектура: окружение
    │   └── 1.3.Operations/      # Операции: взаимодействие
    ├── 2.TargetSystem/          # Целевая система
    │   ├── 2.1.Meaning/         # Смысл: требования
    │   ├── 2.2.Architecture/    # Архитектура: система
    │   └── 2.3.Operations/      # Операции: реализация
    └── 3.CreationSystem/        # Система создания
        ├── 3.1.Meaning/         # Смысл: принципы
        ├── 3.2.Architecture/    # Архитектура: инструменты
        └── 3.3.Operations/      # Операции: методология
```

## Использование

1. **Создайте документы** в соответствующих папках по структуре SRD
2. **Используйте YAML frontmatter** для метаданных документов
3. **Следуйте шаблонам** из каждой папки
4. **Интегрируйте с FPF** для работы с первопринципами

## Принципы работы

- **Минимализм**: создавайте только необходимые документы
- **Практичность**: фокус на применимых знаниях
- **Системность**: связывайте документы между собой
- **Эволюция**: развивайте структуру по мере необходимости

## Начало работы

1. Скопируйте структуру в свой проект
2. Заполните `.fpf/INDEX.md` ключевыми принципами
3. Создайте документы в `content/` по актуальным темам
4. Используйте `CLAUDE.md` для работы с AI-ассистентом
